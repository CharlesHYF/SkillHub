# SkillHub 设计系统

本文件是 SkillHub 前端的视觉与交互契约，M1 起所有界面都据此实现，保证跨屏一致。令牌定义在 `src/index.css`。

## 定位

- 类型：产品 UI（工具型），设计服务于任务，不喧宾夺主。参照 Linear / Raycast / Stripe 一类专业开发者工具的克制质感。
- 原型：`prototype/` 下 7 屏为交互与信息架构的事实来源；配色由本文件补充（原型为黑白）。

## 配色策略：Restrained（克制）

单一品牌强调色，只用于主操作、当前项、链接、焦点；其余一律中性灰。状态用语义色。这是产品 UI 的下限，不做多品牌色堆叠。

硬约束：
- 禁用紫色与任何紫色系。
- 禁用任何渐变（背景/文字/描边）。
- 强调色不用于装饰或非活动态。
- 正文对比度 ≥ 4.5:1，大字/UI 元件 ≥ 3:1（WCAG AA）。

## 令牌（OKLCH，亮暗双主题，见 `src/index.css`）

| 令牌 | 角色 |
|------|------|
| `--sh-bg` | 应用底色（近白微青 / 暗青灰近黑） |
| `--sh-surface` | 卡片、面板、弹层 |
| `--sh-fg` | 正文墨色 |
| `--sh-muted` | 次要文字、占位、说明 |
| `--sh-border` | 发丝边框、分隔线 |
| `--sh-brand` | 品牌强调（青绿 teal，hue~190）：主操作/当前项/链接/图标高亮 |
| `--sh-brand-strong` | 强调色 hover/pressed |
| `--sh-brand-tint` | 当前项/选中行的轻染背景 |
| `--sh-on-brand` | 品牌底上的前景文字 |
| `--sh-ok` | 已同步 / 在线（绿） |
| `--sh-warn` | 部分同步 / 待同步（琥珀） |
| `--sh-danger` | 同步失败 / 离线（红） |

为什么是青绿：SkillHub 的核心动作是把 Skill/MCP 统一、连接、同步到各 Agent —— teal 表达连接与在线，且避开被禁的紫色与最套路的默认蓝。

## 用色规则

- 中性灰微微偏青（chroma≈0.006-0.024，hue~210），与品牌青绿呼应，不做无脑暖/冷调。
- 状态色只用于状态：同步状态徽标、在线点、结果标记。不要拿状态色当装饰。
- Skill / MCP 两类资源：v1 用中性徽标 + 文字（Skill / MCP）区分，如原型；若后续要色分类，用克制的低饱和分类色（禁紫），仅作用于小徽标，不升级为第二品牌色。
- 不用侧边色条（border-left/right 加粗着色）；标记状态用整框发丝边、背景轻染或前导图标。
- 透明度是设计气味：当前项背景用显式 `--sh-brand-tint`，不用 alpha 叠色。

## 排版

- 字体：Inter（拉丁）+ 系统 CJK（PingFang SC / 微软雅黑）；UI 用一套无衬线足矣，最多再加等宽用于代码/路径。
- 固定 rem 字号阶（非流体 clamp）；层级靠字号 + 字重对比（阶比 1.125-1.2）。
- 正文行长 65-75ch；表格可更密。

## 明暗主题

- 暗色不是亮色反相：深度靠面的亮度递增（surface 比 bg 亮），不靠阴影；强调色略降饱和、提亮。
- 默认跟随系统（`prefers-color-scheme`），用户可切换（`ThemeProvider` 在根元素写 `data-theme`）。

## 动效

- 150-250ms，只表达状态（切换/反馈/加载/展开），不做页面级编排入场。
- ease-out 指数曲线，无 bounce/elastic；尊重 `prefers-reduced-motion`。
